# Voidscape Raspberry Pi Notes

Raspberry Pi hosting is not the recommended path for public Voidscape worlds. Use `docs/DEVELOPMENT.md`, `docs/CONFIG-MATRIX.md`, and `docs/OPERATIONS.md` for supported development and deployment flows.

For local LAN experimentation on sufficiently capable hardware, build on a development machine first, then run the server with:

```bash
scripts/run-server.sh
```

Keep public launch, account, and deployment testing on the standard Voidscape infrastructure described in `docs/OPERATIONS.md`.
